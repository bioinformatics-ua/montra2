The plugin structure

The following sections provide some basic directions on how to develop a plugin and the absolute essentials for it to work on the {{config.brand}} plugin platform.

Basic Example

Fundamentally, a plugin for {{config.brand}} is very simple. Only two mandatory things are needed in it's structure, a confs dictionary and a plugin function that receives a self-referencing 'sdk' parameter.

Below follows the typical hello_word plugin.


confs = {
    'name': 'Test plugin'
};
hasContent = function(sdk){
    var fp = FingerprintProxy.getInstance();
    var store = fp.getStore();
    return [store.hasDefaultContent()];
};
plugin = function(sdk){
    sdk.html('Hello world');
    sdk.refresh();
};

The confs dictionary

The confs dictionary is the place that stores the specifications of the widget configurations. Settings such as the display name, the widget icon, and the preferred position and size are defined in this array. Next follows a list of all the available parameters.

name default: "New Plugin"

A string that defines the display name for the plugin.

icon default: undefined

A string that defines the icon for the plugin.

extracss default: []

An array of strings, defining some extra css URLs that should be loaded to execute the plugin.

extralibs default: []

An array of strings, defining some extra javascript library URLs that should be loaded to execute the plugin.

Global widget specific configurations:

width default: 2

Defines the preferred width for the plugin.

height default: 1

Defines the preferred height for the plugin.

x default: 1

Defines the preferred x coordinate for the plugin.

y default: 1

Defines the preferred y coordinate for the plugin.

Fingerprint widget specific configurations:

pos default: next available

Integer that defines the tab position of the plugin.

The hasContent function

The hasContent function is used to find out if the database has, or not, useful content for the specific plugin.

In order to specify with type(s) of content the plugin use, this function must instanciate a DataStore (detailed on the section "Available objects"). Then, the proper has<ContentType>() functions must be called, those methods belongs to the DataStore.

The plugin function

The plugin function is the heart of the plugin. This has access to a series of objects (detailed on the section "Available objects").

Moreover, it is possible to use jquery, bootstrap 2.3.2 styling and font-awesome 4.3 icons. There is also a promise polyfill to browsers than don't support it.

If there is a need to use other external libraries, not included in the list, they can be encompassed through the confs settings "extralibs" and "extracss", already approchead above.
